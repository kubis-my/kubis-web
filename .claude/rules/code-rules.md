# Code Rules

Rules for developers and AI agents working on this codebase.

## Core Principles

- **KISS/YAGNI/SOLID** - Keep it simple, don't add what you don't need, follow solid principles
- **No over-engineering** - Only make changes that are directly requested or clearly necessary
- **Read before modify** - Never propose changes to code you haven't read
- **Follow existing patterns** - Check existing codebase for similar implementations to maintain consistency and avoid redundancy

## Development Workflow

1. **Research existing code** - Check for similar implementations in the codebase first
2. **Plan first** - Output a short plan (no code) and wait for approval
3. **Code second** - Write code only after receiving OK, following existing patterns

## Code Formatting

### Vertical Spacing

- One blank line between different logical operations
- One blank line before and after block comments
- One blank line between variable declarations and operations
- No blank line between closely related operations
